## Core Call Settings

### Call Transfer Number

The phone number or SIP URI that your AI bot will transfer calls to when needed.

<Tabs>
  <Tab title="Phone Number">
    **Standard Phone Number Format**
    
    Enter the phone number without the `+` prefix:
    
    ```
    12185262990
    ```
    
    <Note>
      The system will automatically format the number with the appropriate country code during call processing.
    </Note>
  </Tab>
  
  <Tab title="SIP URI">
    **SIP URI Format**
    
    For SIP-based transfers, use the standard SIP URI format:
    
    ```
    sip:username@domain.com
    sip:12185262990@sip.provider.com
    ```
    
    <Warning>
      SIP URIs must start with `sip:` to be recognized by the system.
    </Warning>
  </Tab>
  
  <Tab title="Custom Configuration">
    **Custom Trunk and DID**
    
    For advanced setups with custom trunks and Direct Inward Dialing (DID):
    
    - **Open field configuration** allows flexible routing
    - **Custom trunk integration** for enterprise solutions
    - **DID mapping** for specific number routing
    
    <Tip>
      Contact [support@vodex.ai](mailto:support@vodex.ai) for custom trunk configuration assistance.
    </Tip>
  </Tab>
</Tabs>

### Disconnect Call Automatically

Automatically disconnect calls after a specified duration to prevent misuse and control costs.

#### **Why Use Auto-Disconnect?**

<AccordionGroup>
  <Accordion title="Prevent Bot Abuse">
    **Protection Against Misuse**
    - Stops people from playing with your bot indefinitely
    - Prevents unnecessary resource consumption
    - Maintains professional call handling
  </Accordion>
  
  <Accordion title="Budget Control">
    **Cost Management**
    - Limits maximum call duration charges
    - Prevents runaway costs from stuck calls
    - Ensures predictable billing
  </Accordion>
  
  <Accordion title="System Efficiency">
    **Resource Optimization**
    - Frees up system resources for legitimate calls
    - Maintains optimal performance
    - Reduces server load
  </Accordion>
</AccordionGroup>

#### **Configuration Options**

| Duration | Use Case | Recommended For |
|----------|----------|-----------------|
| **2-3 minutes** | Quick surveys, confirmations | Simple interactions |
| **5-7 minutes** | Customer service, support | Standard conversations |
| **10-15 minutes** | Sales calls, consultations | Complex discussions |
| **Custom** | Specialized use cases | Enterprise needs |

## Voicemail Detection

Advanced voicemail detection system to optimize call handling and costs.

### How It Works

<Steps>
  <Step title="VAD-Based Detection">
    **Voice Activity Detection Technology**
    - Non-LLM based detection system
    - Works across all countries and languages
    - Independent of ASR, LLM, and voice settings
    - **85% accuracy rate** in standard conditions
  </Step>
  
  <Step title="Smart Timing">
    **Optimized Detection Timing**
    - Bot starts speaking after **10 seconds** of voicemail detection
    - Allows for various voicemail greeting lengths
    - Reduces false positives from long greetings
  </Step>
  
  <Step title="Beep Detection">
    **Enhanced Accuracy with Beep Recognition**
    - Detects voicemail beep signals
    - Restarts message delivery after beep detection
    - Ensures **90% voicemail drop success rate**
    - Maximizes message delivery effectiveness
  </Step>
</Steps>

### Cost Structure

| Scenario | Charge | Explanation |
|----------|--------|-------------|
| **Voicemail Detected + Disconnected** | **3 cents** | Call ends without leaving message |
| **Voicemail Detected + Message Left** | **3 cents** | Successful voicemail delivery |
| **Live Person Answered** | **Standard rate** | Normal conversation charges apply |

<Info>
  **Cost Optimization:** The 3-cent voicemail charge is significantly lower than full conversation rates, helping you manage costs while maintaining campaign effectiveness.
</Info>

## Caller ID Configuration

### Twilio Caller ID Management

<Warning>
  **Twilio Exclusive Feature:** Caller ID configuration is only available when using Twilio as your gateway provider.
</Warning>

#### **How Twilio Caller ID Works**

Twilio's Caller ID system allows you to display specific phone numbers to recipients, improving answer rates and call legitimacy.

#### **Configuration Requirements**

<Steps>
  <Step title="Phone Number Verification">
    **Verify Your Numbers**
    - All caller ID numbers must be verified with Twilio
    - Verification prevents spoofing and ensures compliance
    - Process typically takes 24-48 hours
  </Step>
  
  <Step title="Compliance Setup">
    **Regulatory Compliance**
    - Follow local telecommunications regulations
    - Ensure proper business registration
    - Maintain accurate caller identification
  </Step>
  
  <Step title="Number Pool Management">
    **Optimize Number Usage**
    - Rotate between verified numbers
    - Match caller ID to campaign type
    - Monitor number reputation and performance
  </Step>
</Steps>

#### **Best Practices**

<Tip>
  **Caller ID Strategy:** Use local numbers for regional campaigns, toll-free numbers for national campaigns, and business numbers for professional services.
</Tip>

| Campaign Type | Recommended Caller ID | Benefits |
|---------------|----------------------|----------|
| **Local Business** | Local area code numbers | Higher answer rates, local trust |
| **National Service** | Toll-free numbers | Professional appearance, no charges to recipients |
| **Customer Service** | Main business line | Brand recognition, established trust |
| **Sales Outreach** | Dedicated sales numbers | Call tracking, performance measurement |
